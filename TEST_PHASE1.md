# Phase 1 Bug 修复 — 测试方法

## 前提

```bash
npm run dev
```

打开浏览器访问开发服务器地址。

---

## 测试 1：托盘在移载机/叉车上可见

**步骤：**

1. 点击「🔧 测试」按钮，自动搭建测试回路
2. 点击「▶ 启动」运行仿真
3. 观察画布上托盘的移动轨迹：

**验证点：**

| 阶段 | 预期表现 |
|------|---------|
| 发生器产生托盘 | 托盘出现在发生器（绿色「上料」）处 |
| 托盘进入输送机1 | 托盘沿水平输送机向右移动 |
| 托盘进入移载机 | 托盘**停在移载机中心位置**，等待处理时间结束（**之前会消失，现在应可见**） |
| 托盘进入输送机2 | 托盘沿垂直输送机向下移动 |
| 托盘到达消费者 | 托盘**出现在消费者（蓝色「下料」）处后消失**（**之前不会显示**） |

> 关键验证：托盘在移载机上处理时不再从画布消失。

---

## 测试 2：移载机/叉车选中高亮

**步骤：**

1. 确保仿真已停止（点击「■ 停止」）
2. 工具栏切换到选择模式（第一个工具）
3. 分别点击：绿色「上料」叉车 → 水平输送机 → 移载机 → 蓝色「下料」叉车

**验证点：**

- 点击叉车 → 边框变红变粗
- 点击移载机 → 边框变红变粗
- 点击输送机 → 边框变红变粗（原有功能）

> 关键验证：三种组件类型都能看到红色高亮框。

---

## 测试 3：连线操作可撤销

**步骤：**

1. 清空画布（刷新页面，或手动删除所有组件）
2. 拖入两个链式输送机
3. 切换到连线工具（或按 S 键）
4. 点击第一个输送机的输出端口（绿色圆点）→ 出现绿色预览线
5. 点击第二个输送机的输入端口 → 连线建立
6. 按 **Ctrl+Z** 撤销

**验证点：**

- 连线被撤销，两个输送机之间不再有连线
- 再次按 Ctrl+Z 继续回退之前的操作

---

## 测试 4：内存泄漏（可选，需要 DevTools）

**步骤：**

1. 打开浏览器 DevTools → Memory 面板
2. 选中一个叉车，在属性面板反复切换「波动范围」下拉框（0% → 10% → 20% → 30% → 50% → 0%……）
3. 循环操作 20-30 次

**验证点：**

- Memory 面板的 JS heap size 不会持续单调增长（GC 后可回落）

> 不强制测试，主要靠代码审查确认（draw() 现在先 destroy 旧 Text 再创建新的）。

---

## 快速全量验证

按顺序执行以下操作，全部通过即 Phase 1 验证完成：

```
1. 点击「测试」→ 启动仿真 → 观察托盘在移载机处可见 ✅
2. 停止仿真 → 分别点击三种组件 → 均有红框 ✅
3. 删除连线（点击连线再按 Delete）→ 重新用连线工具连一根线 → Ctrl+Z 线消失 ✅
```
