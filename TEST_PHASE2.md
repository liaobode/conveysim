# Phase 2 类型安全 & 端口语义 — 测试方法

## 前提

```bash
npm run dev
```

---

## 测试 1：消费者端口语义正确

**步骤：**

1. 清空画布（刷新页面）
2. 从工具栏拖入一个「链式输送机」
3. 从工具栏拖入一个「叉车-消费者」到输送机输出端口附近
4. 观察是否自动连线

**验证点：**

- 自动连线方向为 **输送机 output → 消费者 input**（而不是消费者 output → 输送机）
- 切换到连线工具，悬停在消费者上 → 端口标签应显示为 **input**（绿色圆点位于输出端位置）

---

## 测试 2：测试回路可正常运行

**步骤：**

1. 点击「🔧 测试」按钮
2. 点击「▶ 启动」运行仿真
3. 观察托盘流动

**验证点：**

- 托盘从发生器出发，依次经过输送机1 → 移载机 → 输送机2 → 消费者
- 消费者正确接收托盘
- 吞吐量数据正常更新

---

## 测试 3：连线方向自动反转

**步骤：**

1. 清空画布
2. 拖入一个链式输送机
3. 切换到连线工具（按 S 键）
4. 先点击**消费者**的 input 端口（作为起点）
5. 再点击输送机的 output 端口（作为终点）

**验证点：**

- 连线成功建立
- 连接方向自动反转为 **输送机 → 消费者**（数据流向正确）

---

## 测试 4：拖拽叉车/移载机无异常

**步骤：**

1. 拖入一个叉车
2. 选中它，拖动到新位置

**验证点：**

- 拖动正常，组件跟随鼠标移动
- 松开后位置更新
- 浏览器控制台无类型错误

---

## 快速验证清单

```
1. 搭测试回路 → 启动 → 托盘正常流动 ✅
2. 拖入消费者靠近输送机输出端 → 自动连线方向正确 ✅
3. 手动连线消费者 input → 输送机 output → 方向反转正确 ✅
4. 拖拽叉车/移载机移动 → 无异常 ✅
```
